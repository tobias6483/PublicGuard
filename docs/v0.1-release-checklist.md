# v0.1 Release Checklist

Use this checklist before tagging or publishing a v0.1 PublicGuard release. The
goal is an honest beta-quality release: local-first, unsigned unless stated
otherwise, with hardware behavior validated or clearly marked as untested.

## Release Scope

v0.1 should focus on the local protection loop:

- Arm/Disarm with Touch ID or password.
- Charger, Wi-Fi, sleep/wake, Bluetooth proximity, and idle triggers.
- Loud and silent responses.
- Optional lock screen action.
- Local-only event log with minimal and encrypted storage options.
- Menu bar diagnostics for recent events and trigger state.
- Unsigned app artifact for maintainer QA.

Keep these out of v0.1 unless a separate privacy/security review says
otherwise:

- Camera snapshots.
- Microphone behavior.
- Cloud sync.
- Analytics or tracking.
- Hidden location collection.
- Signed/notarized public distribution claims.

## Before Tagging

- [ ] Start from an up-to-date `main`.
- [ ] Confirm [v0.1-release-notes.md](v0.1-release-notes.md) has current
      hardware QA status and verification results.
- [ ] Confirm `CHANGELOG.md` has a concise v0.1 summary or release note entry.
- [ ] Confirm [README.md](../README.md) describes implemented behavior without
      overstating lid-close or Bluetooth reliability.
- [ ] Confirm [docs/hardware-qa-results.md](hardware-qa-results.md) has current
      hardware QA results or explicitly leaves untested areas as `Not tested`.
- [ ] Confirm [docs/requirements.md](requirements.md) matches the intended
      v0.1 feature coverage.
- [ ] Confirm [docs/camera-snapshot-privacy-review.md](camera-snapshot-privacy-review.md)
      still marks camera snapshot behavior out of scope.

## Local Verification

Run these from the repository root:

```sh
scripts/release_check.sh
```

The script runs `swift build`, `swift test`, `scripts/build_app.sh`,
`scripts/validate_app_bundle.sh`, and an unsigned artifact dry run with a
SHA-256 checksum. Record the exact commit and results in the release PR or
release notes.

## Manual Hardware QA Gate

Use [hardware-qa-results.md](hardware-qa-results.md) for the release pass. A
v0.1 beta can still ship with partial hardware coverage, but the release notes
must state what was not tested.

Minimum recommended pass:

- [ ] Charger disconnect.
- [ ] Wi-Fi SSID change.
- [ ] Wi-Fi disconnect with `Ignore Wi-Fi Disconnects` off and on.
- [ ] Sleep/wake with lid open.
- [ ] Lid close/reopen on MacBook hardware.
- [ ] Bluetooth proximity with at least one real target device.
- [ ] Idle timeout with disabled and positive timeout settings.
- [ ] Notifications from `dist/PublicGuard.app`.
- [ ] Lock screen action enabled and disabled.
- [ ] Launch at login from `dist/PublicGuard.app`.
- [ ] Bundled alarm sounds and normal/maximum app playback volume.
- [ ] Plain text and encrypted event log storage.

No-go results for v0.1:

- Disarm or alarm stop bypasses Touch ID/password.
- Lock screen setting is ignored.
- Minimal log detail records SSIDs or Bluetooth device names in new entries.
- Encrypted log storage writes plaintext event details to `events.log.enc`.
- README or release notes imply guaranteed live alarm playback while a sleeping
  MacBook lid remains closed.

## README Media

Add screenshots or a short GIF only after the app bundle UI has been manually
checked. Suggested captures:

- Menu bar closed state with PublicGuard visible.
- Armed menu with active preset/status summary.
- Settings submenu showing presets and trigger controls.
- Recent Trigger Status diagnostics.
- Recent Events menu preview with non-sensitive sample data.

Avoid publishing images that reveal personal SSIDs, Bluetooth device names,
local file paths, or notification contents with private information.

## GitHub Artifact Dry Run

Before a public release, run the `App Artifact` workflow from GitHub Actions or
from a temporary version tag. Confirm the uploaded artifact includes:

- `PublicGuard.app.zip`
- `PublicGuard.app.zip.sha256`
- A validated unsigned `PublicGuard.app`

The artifact is unsigned and not notarized. Release notes must say that clearly.
