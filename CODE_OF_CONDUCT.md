# Code of Conduct

PublicGuard is a small open-source project for people who want safer public
workspace habits without giving up local control or privacy. The project should
be welcoming to contributors, testers, and users with different experience
levels.

## Expected Behavior

- Be respectful, patient, and constructive.
- Assume good intent when asking for clarification.
- Keep technical disagreement focused on the work.
- Explain tradeoffs clearly when reviewing code or product ideas.
- Respect privacy and security concerns, especially around permissions,
  logging, Bluetooth, networking, authentication, and local device behavior.
- Help newcomers find the right documentation, issue template, or test path.

## Unacceptable Behavior

- Harassment, insults, threats, or sustained personal attacks.
- Sexualized language or attention.
- Discriminatory language or conduct.
- Publishing private information without consent.
- Pressuring maintainers or contributors to disclose private security details
  in public.
- Repeatedly derailing issues, pull requests, or discussions after maintainers
  ask for the thread to return to scope.

## Enforcement

Maintainers may remove comments, close issues, block users, or limit
participation when behavior harms the project or other participants. Enforcement
should be proportional, documented when practical, and focused on keeping the
project useful and safe.

## Reporting

For conduct concerns, open a GitHub issue if the report can be public. If the
concern includes private security details, follow [SECURITY.md](SECURITY.md)
instead.

This policy can be updated as the project grows and gains a private maintainer
contact path.
